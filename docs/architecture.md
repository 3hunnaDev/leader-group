# Архитектура проекта

Проект организован по слоям:

- `app` — корневая инициализация приложения, провайдеры, роутинг, layout.
- `pages` — страничные композиции, собирают `widgets`.
- `widgets` — крупные самостоятельные UI-блоки страницы.
- `features` — пользовательские сценарии и бизнес-действия.
- `entities` — доменные сущности и их модели.
- `shared` — переиспользуемые утилиты, конфиги и базовые UI-компоненты.

## Правила зависимостей

- Допустимое направление: `app -> pages -> widgets -> features -> entities -> shared`.
- Слой не импортирует вышестоящие слои.
- Каждый модуль должен иметь явный публичный API через `index.ts`.
- Границы слоев проверяются через ESLint правило `no-restricted-imports` и для относительных импортов, и для алиасов (`@app/*`, `@pages/*`, ...).

## Текущий статус

- `HomePage` собирается из виджетов в `widgets/home/*`.
- `SiteHeader` выделен в отдельный widget с `model`-слоем.
- Общая модалка остается в `shared/ui/modal`.
- `features` и `entities` подготовлены как точки роста.
