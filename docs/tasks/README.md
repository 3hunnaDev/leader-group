# Tasks

Здесь хранится backlog задач для следующего агента и команды.

## Правила ведения

- Все новые задачи добавляются в `backlog.md`.
- У задачи должен быть понятный идентификатор, статус, контекст и критерии готовности.
- Если задача закрыта, её не удаляют: переносят в секцию `Done` с датой.
- Если задача меняет архитектурное направление, вместе с ней обновляется соответствующий ADR.
- Если задача приводит к заметному изменению проекта, после выполнения обновляется `CHANGELOG.md`.

## Рекомендуемый формат

- ID
- Status
- Priority
- Title
- Context
- Scope
- Acceptance Criteria
- Verification
